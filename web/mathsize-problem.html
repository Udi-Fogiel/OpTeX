LuaTeX format with extended plainTeX macros

Bug in luaotfload version 3.24 2023-08-19

Unfortunately, OpTeX depends on the package luaotfload which is managed by LaTeX team. They are copying most of the code from ConTeXt (by Hans Hagen) and they add a few additional features.

The luaotfload package is used by OpTeX for loading Unicode fonts in OpenType format.

The version 3.24 2023-08-19 of the luaotfload package canceled the support of mathsize font feature, but OpTeX math typesetting is based on this feature. The result is that script size and scriptscript sizes are not smaller than text font size, i.e. the Unicode math typesetting is completely broken. See also the discussion about this at the luaoftload project page.

What you can do: you can apply the patch in OpTeX or the patch of luaotfload or you can downgrade the luaotfload package to a previous version (before 3.24).

And you can wait for a new release of the luaotfload package...
